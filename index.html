  • Ingress & Load Balancing: Set up Nginx Ingress Controller to handle traffic routing and load balancing for the GoLang app.
  • Continuous Delivery with ArgoCD: Configured ArgoCD to automatically sync and deploy changes from GitHub to EKS, ensuring continuous delivery and consistency across environments.
  • DNS Configuration: Used AWS Route 53 to map a custom DNS to the deployed application for easy access.
Challenges Overcome:
  • Docker Image Architecture: Resolved by rebuilding the Docker image with the correct architecture.
  • golangci-lint Issues: Fixed Go version compatibility and ensured the linter worked smoothly in the CI/CD pipeline.
  • CI/CD Automation: Fully automated the CI/CD pipeline with GitHub Actions to build, tag, and deploy the application using Helm and ArgoCD.
Outcome:
  • The application was successfully deployed to AWS EKS with an automated CI/CD pipeline powered by GitHub Actions, Helm, and ArgoCD. The deployment process became efficient, scalable, and fully automated, with continuous monitoring for updates and changes.

Amazon Prime Video

Project Overview:
I successfully designed, deployed, and automated a production-grade Amazon Prime Video Clone using Terraform, Jenkins, ArgoCD, Kubernetes, and AWS services. This deployment runs on Amazon EKS version 1.33, with centralized encryption control and secret protection powered by AWS KMS.

Key Contributions:
  • Source Control: Application code and Kubernetes/Helm manifests managed in GitHub.
  • CI Pipeline: Jenkins automates builds, npm installs, SonarQube code analysis, and Trivy security scans. Memory issues were resolved by upgrading the instance to t2.large and using cleanWs().
  • Containerization: Application services containerized using Docker and pushed to AWS ECR, resolving initial AWS CLI credential issues.
  • Infrastructure Provisioning: Entire AWS infrastructure provisioned via Terraform, including VPC, subnets, route tables, Internet Gateway, security groups, IAM roles, KMS, CloudWatch, EKS v1.33, ECR, and Route 53 for DNS management.
  • EKS Node Creation: Initially failing or taking 30+ minutes due to IAM misconfigurations, KMS alias conflicts, Elastic IP limits, and duplicate CloudWatch logs. Fixed by attaching required IAM policies, cleaning unused EIPs, resolving KMS alias conflicts, and removing duplicate log groups â†’ node creation succeeded in 8â€“10 minutes.
  • GitOps Deployment: Jenkins pipelines with Helm installed ArgoCD, Prometheus, and Grafana onto EKS. ArgoCD automatically syncs manifests from GitHub for continuous deployment.
  • Route 53 Integration: Configured public hosted zones, routing rules, and linked ELBs to provide high availability and secure traffic routing. Health checks and failover ensure traffic goes only to healthy nodes.
  • Security: AWS KMS used for encryption; aws-auth ConfigMap updated to give Jenkins proper RBAC (e.g., eks:DescribeCluster, eks:ListClusters) permissions for EKS deployments.
  • Monitoring & Observability: Prometheus collects metrics; Grafana dashboards provide real-time insights into cluster and application performance.
